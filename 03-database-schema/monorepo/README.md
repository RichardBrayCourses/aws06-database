# Lesson 03 - Database Schema

This lesson adds Flyway database migrations to the RDS-backed application.

The schema creates:

- `registered_user`
- `nickname` on `registered_user`
- `images`
- indexes for gallery search by title, description, and nickname

The application functionality is otherwise unchanged in this lesson.

## Run

From this folder:

```bash
pnpm install
pnpm run deploy-everything
```

`deploy-everything` deploys RDS, runs Flyway migrations, deploys the API, and publishes the UI.

To migrate the database independently:

```bash
pnpm run database:migrate
```

To reset and then recreate the schema:

```bash
pnpm run database:reset
pnpm run database:migrate
```

To destroy everything:

```bash
pnpm run destroy-everything
```

## Connecting With pgAdmin

Read the RDS credentials from Secrets Manager:

```bash
SECRET_ARN=$(aws ssm get-parameter --name /rds/secret-arn --query "Parameter.Value" --output text)
aws secretsmanager get-secret-value --secret-id "$SECRET_ARN" --query "SecretString" --output text
```

Use the returned `host`, `port`, `username`, and `password` in pgAdmin.

The database name is:

```text
uptickart
```

## Code Changes In This Lesson

This lesson adds the first database schema and a script for running Flyway migrations against the RDS cluster.

The migration files live in `database/sql`. Flyway runs them in version order, so the file names are part of the design:

```text
V1__Create_users_table.sql
V2__Add_nickname_to_registered_user.sql
V3__Create_images_table.sql
V4__Add_indexes_for_gallery_search.sql
```

The first migration creates the `registered_user` table. The `sub` column stores the Cognito user ID, and it is unique because one Cognito user should only have one row:

```sql
CREATE TABLE IF NOT EXISTS registered_user (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    sub VARCHAR(255) NOT NULL UNIQUE,
    email VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

The second migration adds an optional nickname:

```sql
ALTER TABLE registered_user
ADD COLUMN nickname VARCHAR(20) NULL;
```

The third migration creates the `images` table. Each image belongs to a registered user through the `sub` foreign key:

```sql
CREATE TABLE IF NOT EXISTS images (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    sub VARCHAR(255) NOT NULL,
    uuid_filename VARCHAR(36) NOT NULL UNIQUE,
    image_name VARCHAR(40) NOT NULL,
    image_description VARCHAR(120) NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_images_sub
        FOREIGN KEY (sub)
        REFERENCES registered_user(sub)
        ON DELETE CASCADE
);
```

The fourth migration adds indexes for the fields the gallery will search later:

```sql
CREATE INDEX IF NOT EXISTS idx_images_image_name ON images(image_name);
CREATE INDEX IF NOT EXISTS idx_images_image_description ON images(image_description);
CREATE INDEX IF NOT EXISTS idx_registered_user_nickname ON registered_user(nickname);
```

The migration runner is `scripts/src/database-migrate.ts`. It reads the RDS secret ARN from SSM, fetches the database credentials from Secrets Manager, and then runs Flyway:

```ts
const secretArn = await getParameter("/rds/secret-arn");
const secretValue = await secretsClient.send(
  new GetSecretValueCommand({ SecretId: secretArn }),
);
```

Before Flyway runs, the script connects to the default `postgres` database and creates the course database if it does not exist:

```ts
const result = await adminClient.query(
  "SELECT 1 FROM pg_database WHERE datname = $1",
  [databaseName],
);

if (result.rows.length === 0) {
  await adminClient.query(`CREATE DATABASE ${databaseName}`);
}
```

The Flyway command is built from the RDS credentials and the local `database/sql` folder:

```ts
const jdbcUrl = `jdbc:postgresql://${credentials.host}:${credentials.port ?? 5432}/${databaseName}?sslmode=require`;

execSync(
  `flyway -connectRetries=${FLYWAY_RETRIES} -url="${jdbcUrl}" -user="${credentials.username}" -password="${credentials.password}" -locations="filesystem:${sqlDir}" ${cleanDisabledFlag} ${flywayCommand}`.trim(),
  { stdio: "inherit", cwd: rootDir },
);
```

The root package scripts now include:

```json
"database:migrate": "tsx scripts/src/database-migrate.ts",
"database:reset": "tsx scripts/src/database-migrate.ts reset"
```

`database:migrate` applies pending migrations. `database:reset` runs `flyway clean`, which drops the schema objects so the migrations can rebuild them from scratch.
